# Schema Comparison: Database vs WordPress Plugin vs React App

## 🔍 Issue Identified

The TypeScript types file still uses the **old table name** `quote_requests` (plural), but everything else uses `quote_request` (singular) as of v1.1.0.

---

## 📊 Current Schema Comparison

| Column | Type | Nullable | Default | Notes |

### What WordPress Plugin Expects

**File**: `wordpress-plugin/forge-dashboard/includes/class-quote-manager.php`

The plugin queries the table as `quote_request` (singular) and expects these columns:

- ✅ `id` - Used for primary key lookups
- ✅ `name` - Customer name
- ✅ `email` - Customer email
- ✅ `phone` - Customer phone (nullable)
- ✅ `company` - Customer company (nullable)
- ✅ `quantity` - Order quantity
- ✅ `material` - Material type
- ✅ `timeline` - Expected timeline (nullable)
- ✅ `notes` - Additional notes (nullable)
- ✅ `model_data` - JSON string containing:
  - `quoteId` - Quote reference ID (e.g., HF-123456789)
  - `fileName` - Original 3D model filename
  - `material` - Material name
  - `quantity` - Quantity (redundant with top-level)
  - `pricing` - Object with pricing breakdown
    - `baseCost`
    - `materialCost`
    - `finishingCost`
    - `quantityDiscount`
    - `total`
  - `attachmentUrl` - URL to uploaded 3D model file
  - `dimensions` - 3D model dimensions
  - `vertices` - Vertex count
  - `triangles` - Triangle count
- ✅ `status` - Quote status (pending, processing, quoted, etc.)
- ✅ `created_at` - Timestamp

**Not used by WordPress plugin but in schema**:
- `user_id` - Not used (quotes can be anonymous)
- `model_id` - Not used (file URL stored in model_data instead)

## ❌ The Bug

**File**: `services/supabase/types.ts` (Line 110)

```typescript
quote_requests: {  // ❌ WRONG - Should be quote_request
  Row: { ... },
  Insert: { ... },
  Update: { ... }
}
```

**But the hook uses**: `services/supabase/hooks.ts` (Line 242)

```typescript
const { data, error } = await supabase
  .from('quote_request')  // ✅ CORRECT - singular
  .insert(quote)
```

### Why This Causes Issues

1. **Type mismatch**: TypeScript types reference `quote_requests` but code uses `quote_request`
2. **Inconsistency**: Types don't match actual database table name
3. **Potential runtime errors**: If code ever relies on the types for table names
4. **Confusion**: Developers see conflicting names in codebase

---

## ✅ The Fix

Update `services/supabase/types.ts` to use `quote_request` (singular):

```typescript
// Line 110: Change from quote_requests to quote_request
quote_request: {
  Row: {
    id: string;
    user_id: string | null;
    model_id: string | null;
    name: string;
    email: string;
    phone: string | null;
    company: string | null;
    quantity: number;
    material: string;
    timeline: string | null;
    notes: string | null;
    model_data: Json;
    status: string;
    created_at: string;
  };
  Insert: {
    id?: string;
    user_id?: string | null;
    model_id?: string | null;
    name: string;
    email: string;
    phone?: string | null;
    company?: string | null;
    quantity: number;
    material: string;
    timeline?: string | null;
    notes?: string | null;
    model_data: Json;
    status?: string;
    created_at?: string;
  };
  Update: {
    status?: string;
    notes?: string | null;
  };
};
```

And update the convenience types (Lines 165, 171):

```typescript
export type QuoteRequest = Database['public']['Tables']['quote_request']['Row'];
export type QuoteRequestInsert = Database['public']['Tables']['quote_request']['Insert'];
```

## 📝 Summary

**The Issue**: Column names are all correct, but the **table name** in `types.ts` is wrong.

- **Database table**: `quote_request` ✅
- **WordPress plugin uses**: `quote_request` ✅
- **React hook uses**: `quote_request` ✅
- **TypeScript types reference**: `quote_requests` ❌ (WRONG)

**The Fix**: Change `quote_requests` to `quote_request` in types.ts (3 places)

This was introduced when we changed the table name from plural to singular in v1.1.0, but forgot to update the TypeScript types file.
